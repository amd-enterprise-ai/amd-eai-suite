# ActionButton

A customizable button component with predefined styling variants that wraps HeroUI's Button component with enforced design system constraints.

## Overview

The `ActionButton` component provides a consistent button interface across the application with three main variants:

- **Primary**: Solid variant for primary actions (blue filled)
- **Secondary**: Flat variant for secondary actions (default, gray flat)
- **Tertiary**: Light variant for tertiary/subtle actions (transparent with hover)

---

## Variants

### Primary Button

Use primary buttons for the main action on a page or in a modal. Uses solid blue styling.

<JSX>
  <ActionButton primary onPress={() => alert("Primary clicked!")}>
    Save Changes
  </ActionButton>
</JSX>

### Secondary Button

Use secondary buttons for less prominent actions. This is the default variant when no variant prop is specified.

<JSX>
  <ActionButton secondary onPress={() => alert("Secondary clicked!")}>
    Cancel
  </ActionButton>
</JSX>

### Tertiary Button

Use tertiary buttons for subtle actions or when you need a lighter visual weight.

<JSX>
  <ActionButton tertiary onPress={() => alert("Tertiary clicked!")}>
    Learn More
  </ActionButton>
</JSX>

### All Variants Side by Side

  </div>

## With Icons

The `icon` prop allows you to add an icon before the button text.

    </ActionButton>
    <ActionButton secondary icon={<IconSettings size={18} />}>

    </ActionButton>

      Edit

---

## Icon Only Buttons

When no children are provided, the button automatically becomes icon-only. Always provide an `aria-label` for accessibility.

<JSX>
  <div className="flex gap-4 items-center">
    <ActionButton primary icon={<IconPlus size={18} />} aria-label="Add" />
    <ActionButton
      secondary
      icon={<IconSettings size={18} />}
      aria-label="Settings"
    />
    <ActionButton tertiary icon={<IconTrash size={18} />} aria-label="Delete" />
  </div>
</JSX>

## Props Reference

### Variant Props (Exclusive)

Only one variant prop can be true at a time:

| Prop        | Type      | Default | Description                                  |
| ----------- | --------- | ------- | -------------------------------------------- |
| `primary`   | `boolean` | `false` | Renders as solid variant (filled background) |
| `secondary` | `boolean` | `false` | Renders as flat variant (subtle background)  |
| `tertiary`  | `boolean` | `false` | Renders as light variant (transparent)       |

_If no variant is specified, defaults to secondary._

### Common Props

| Prop           | Type                                                                          | Default | Description                                 |
| -------------- | ----------------------------------------------------------------------------- | ------- | ------------------------------------------- |
| `size`         | `"sm" \| "md" \| "lg"`                                                        | `"md"`  | Button size and border radius               |
| `color`        | `"primary" \| "secondary" \| "success" \| "warning" \| "danger" \| "default"` | varies  | Button color theme                          |
| `icon`         | `React.ReactNode`                                                             | -       | Icon to display before button text          |
| `isLoading`    | `boolean`                                                                     | `false` | Shows loading spinner, hides icon           |
| `isDisabled`   | `boolean`                                                                     | `false` | Disables button interaction                 |
| `fullWidth`    | `boolean`                                                                     | `false` | Button expands to fill container            |
| `startContent` | `React.ReactNode`                                                             | -       | Custom content before text (overrides icon) |
| `children`     | `React.ReactNode`                                                             | -       | Button label/content                        |
| `onPress`      | `(e: PressEvent) => void`                                                     | -       | Click/press handler                         |

### Inherited HeroUI Button Props

All other HeroUI Button props are supported except: `variant`, `radius`, `spinner`, `isIconOnly` (these are controlled internally).

## Best Practices

1. **Use Primary for main actions**: Reserve primary buttons for the most important action on a page
2. **Use Secondary for supporting actions**: Cancel, back, or less prominent actions
3. **Use Tertiary for subtle actions**: "Learn more", "View details", or inline actions
4. **Provide aria-labels for icon-only buttons**: Essential for accessibility
5. **Show loading states for async actions**: Provides user feedback
6. **Use danger color for destructive actions**: Clear visual warning for delete/remove operations

---

## Accessibility

- Inherits HeroUI Button accessibility features
- Keyboard navigation support (Tab, Enter, Space)
- ARIA attributes are properly set
- Always provide `aria-label` for icon-only buttons:

```tsx
<ActionButton tertiary icon={<IconEdit />} aria-label="Edit item" />
```
